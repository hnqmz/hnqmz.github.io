Information Maximum GAN

Posted by Changtao Li on 2021-04-08

Background

We know how to generate the specific kind of data we want by using a Conditional GAN. Admittedly, this can let us control some part of the generator. However, this is far from enough: it cannot let us control the details of the data samples generated by the generator. For example, I want to have some thicker handwritten 2s and some thinner handwritten 2s. At this scenario, CGAN will not help. So, how can we control the details of the generated data samples.

We know the latent vectors are, in fact, compressed representations of the data instances. Each element of the latent vectors has some semantic meaning. Intuitively, we can manipulate the latent vector to control the generated samples. However, some studies show that the semantic features are highly entangled. As a result, we should decouple those semantic features before we can use them to control the details of the generated samples.

In 2016, some researchers put up with InfoGAN in their paper: InfoGAN: Interpretable Representation Learning by Information Maximizing Generative Adversarial Nets. This GAN structure gives us more control to the generator.

InfoGAN

Structure

The basic idea is this: along with the random noises provided to the generator, we also give the generator some latent codes, discrete or/and continuous. As for the discriminator, we want it to behave like a normal discriminator. So far, there is nothing different from the traditional GAN. What makes InfoGAN different is that we need an auxiliary network to interpret the latent codes from the input images, fake and real. Here is a picture to illustrate the structure of InfoGAN：
